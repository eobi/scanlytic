Endpoints

API Reference

POST /v1/analyze/message

Analyze a text message for scam indicators using AI and pattern matching.

Request Body

Parameter Type Required Description
content string Yes The message content to analyze (max 5000 chars)
context string No Source context: "whatsapp", "sms", "email", "social"
sender_phone string No Sender's phone number for reputation check

Response

Field Type Description
scam_score integer Risk score from 0-100
verdict string LOW_RISK, MODERATE_RISK, HIGH_RISK, CRITICAL_RISK
threat_type string Type of scam detected (see threat types below)
explanation string Human-readable explanation of the analysis
recommended_action string Suggested action for the user
signals array List of detected scam indicators
request_id string Unique request identifier
POST /v1/analyze/url

Check a URL against phishing databases and reputation services.

Request Body

Parameter Type Required Description
url string Yes The URL to analyze
follow_redirects boolean No Follow redirects and analyze final URL (default: true)
POST /v1/analyze/phone

Verify a phone number for carrier type, VoIP detection, and reputation.

Request Body

Parameter Type Required Description
phone string Yes Phone number in E.164 format (+234...)
POST /v1/analyze/profile

Check if a profile photo or social media account is a potential catfish/fake.

Request Body

Parameter Type Required Description
image_url string No* URL of the profile image to analyze
image_base64 string No* Base64-encoded image data
profile_url string No* Social media profile URL to analyze

* At least one of image_url, image_base64, or profile_url is required.

Security Best Practices

  • Use environment variables for key storage
